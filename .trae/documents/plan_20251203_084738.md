# 实现 Progress Test 组件

## 计划概述

创建一个 Svelte 组件来测试 `/progress_test` API，该组件将使用 EventSource API 接收服务器发送的进度更新，并以可视化方式显示进度。

## 实现步骤

### 1. 创建 ProgressTest 组件

- 在 `src/component/` 目录下创建 `ProgressTest.svelte` 文件

- 实现以下功能：
  - 一个按钮来触发进度测试

  - 一个进度条来可视化进度

  - 一个区域来显示进度消息

  - 使用 EventSource API 来接收服务器发送的进度更新

  - 处理连接关闭和错误情况

### 2. 集成到 App.svelte

- 将 `ProgressTest.svelte` 组件添加到 `App.svelte` 中

- 确保组件在页面上正确显示

## 技术细节

- 使用 EventSource API 处理服务器发送事件 (SSE)

- 使用 Svelte 的响应式系统来更新 UI

- 实现错误处理和连接管理进度

- 进度使用 progress 字段传播（0.0\~1.0）

## 预期效果

- 用户可以点击按钮启动进度测试

- 进度条会随着服务器发送的更新而动态变化

- 显示详细的进度消息

- 测试完成后显示最终结果

## 文件变更

- `src/component/ProgressTest.svelte` (新文件)

- `src/App.svelte` (修改，添加新组件)
